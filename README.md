# دكانجي

نسخة ويب عربية متجاوبة لمنصة طلب من متاجر الحي، مبنية بدون مكتبات خارجية وتعمل كتطبيق ويب تقدمي PWA.

## التشغيل

يتطلب المشروع Node.js فقط:

```powershell
npm start
```

ثم افتح:

```text
http://127.0.0.1:4173
```

## الواجهات الجاهزة

- واجهة العميل: الرئيسية، المتاجر، العروض، صفحة المتجر والمنتج.
- سلة من متجر واحد مع خيارات المنتج والكمية والملاحظات.
- إنهاء الطلب: العنوان، الموعد، طريقة الاستلام، الدفع وسياسة البدائل.
- تسعير توصيل تلقائي للمسافات البعيدة: المسافة ذهاباً وإياباً × سعر الكيلومتر الذي يحدده التاجر.
- إظهار مسافة الطريق وزمن الوصول ورسوم التوصيل مباشرة عند تغيير عنوان العميل.
- حساب العميل وطلبات سابقة مع التتبع وإعادة الطلب.
- لوحة المتجر: الإحصائيات، الطلبات، المنتجات، العروض، البيانات والاشتراك.
- لوحة الإدارة: الإحصائيات، المتاجر، العملاء، الطلبات، الشكاوى والمحتوى.
- تصدير بيانات تجريبية بصيغة CSV.
- حفظ السلة والمفضلة والطلبات محلياً في المتصفح.
- ملف Manifest وService Worker للتثبيت والعمل دون اتصال.

## النشر من GitHub إلى Vercel دون طرفية

### رفع المشروع إلى GitHub

1. افتح GitHub Desktop وسجل الدخول إلى حساب GitHub.
2. اختر `File` ثم `Add local repository`.
3. اختر مجلد المشروع: `C:\projects\New Dukkanci`.
4. إذا ظهر أن المجلد ليس مستودع Git، اختر `Create a repository here`.
5. اكتب ملخصاً مثل `Initial Dukkanci website` ثم اضغط `Commit to main`.
6. اضغط `Publish repository` وحدد ما إذا كان المستودع عاماً أو خاصاً.

### نشر الموقع على Vercel

1. سجل الدخول إلى Vercel بواسطة حساب GitHub.
2. اختر `Add New` ثم `Project`.
3. اختر مستودع `dukkanci` واضغط `Import`.
4. اجعل `Framework Preset` بالقيمة `Other`.
5. اترك `Root Directory` على جذر المشروع، ولا تضف أمر بناء أو مجلد إخراج.
6. اضغط `Deploy`.

بعد النشر، كل تحديث يتم رفعه إلى الفرع `main` في GitHub سيُنشئ نسخة جديدة تلقائياً على Vercel.

## التكاملات الخارجية

### مزامنة حلويات السلطان

توجد أداة مزامنة تجلب الفروع والمنتجات والأسعار والصور من الموقع الرسمي وتحفظها محلياً:

```powershell
node scripts/sync-alsultan.mjs
```

بعد تشغيلها، ارفع التغييرات إلى GitHub ليعيد Vercel نشر الكتالوج المحدث تلقائياً.

### مزامنة حلويات زيتونة

تجلب أداة زيتونة المنتجات المتاحة لتركيا مع الأسعار والأوزان والصور الرسمية، ثم تحفظها محلياً:

```powershell
node scripts/sync-zaitoune.mjs
```

بعد تشغيلها، ارفع الملفات المتغيرة إلى GitHub ليعيد Vercel نشر المتجر تلقائياً.

### خرائط Google

خدمة حساب التوصيل مربوطة من الخادم بـ Google Routes API. فعّل Routes API في مشروع Google Cloud، ثم شغّل التطبيق مع المفتاح في متغير بيئة:

```powershell
$env:GOOGLE_MAPS_API_KEY="ضع_المفتاح_هنا"
npm start
```

لا تضع المفتاح داخل `app.js`. عند عدم توفر المفتاح يستخدم النظام تقديراً مؤقتاً للمسافة، ويتحول تلقائياً إلى مسار Google الحقيقي عند إضافته.

في Vercel افتح المشروع ثم `Settings` ثم `Environment Variables`، وأضف متغيراً باسم `GOOGLE_MAPS_API_KEY`. بعد حفظه أعد نشر آخر نسخة من تبويب `Deployments`.

### خدمات أخرى

الواجهات ونقاط الاستخدام جاهزة، لكن التشغيل الحقيقي للخدمات التالية يتطلب حسابات ومفاتيح سرية:

- WhatsApp Business API لإشعارات الطلبات والتحقق.
- iyzico للدفع واشتراكات المتاجر.
- تسجيل Google.

البيانات الحالية تجريبية، وتُحفظ داخل المتصفح لتسهيل استعراض جميع التدفقات دون إعداد قاعدة بيانات.
