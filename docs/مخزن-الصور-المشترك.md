# مخزن الصور المشترك لمتاجر السوبر ماركت

## الفكرة

عند تأسيس متجر جديد بتصنيف
"سوبر ماركت"،
يستطيع صاحبه تصفّح بنك منتجات جاهز
(صور نظيفة، بلا خلفية، بقياس وزووم موحّد)
ساهمت به متاجر السوبر ماركت الأخرى على
دكانجي،
واستيراد ما يريد منه إلى متجره: يبقى الاسم قابلاً للتعديل،
ولا يُفرض عليه سعر — يمكن تركه بلا سعر الآن وتحديده لاحقاً،
مع إمكانية إضافة سعر عرض فوراً إن رغب.

لا تُعرض أي صورة لمتجر آخر إلا بعد موافقة الإدارة صراحة على أنها خالية من أي علامة أو شعار خاص بمتجر بعينه.

## الملفات التي أُضيفت

| الملف | الغرض |
|---|---|
| `migrations/20260704_shared_supermarket_catalog.sql` | جدول `catalog_products` المشترك + عمود `catalog_product_id` في `products` + صلاحيات القراءة/الكتابة |
| `lib/catalog-image-pipeline.js` | خطوتا المعالجة: إزالة الخلفية وأي علامة متجر عبر OpenAI، ثم توحيد القياس والزووم عبر sharp |
| `api/catalog-process-image.js` | معالجة صورة واحدة يدوياً (استخدام إداري) |
| `api/catalog-review.js` | قائمة المراجعة: الموافقة أو الرفض قبل ظهور أي منتج لبقية المتاجر |
| `scripts/build-shared-catalog.js` | مهمة تعبئة أولى: تجمع منتجات متاجر السوبر ماركت الحالية، تُزيل التكرار، وتعالج الصور دفعة واحدة |
| تعديلات على `app.js` و `styles.css` | زر "استيراد من مخزن الصور" في لوحة المتجر، يظهر فقط لمتاجر السوبر ماركت |

## خطوات التشغيل

1. نفّذ ملف الهجرة `migrations/20260704_shared_supermarket_catalog.sql` على مشروع Supabase.
2. أنشئ حاوية تخزين عامة (Storage bucket) باسم `catalog-images` من لوحة Supabase.
3. تأكد من متغيرات البيئة في Vercel:
   `OPENAI_API_KEY`
   (نفس المفتاح المستخدم في `تحسين الصورة` حالياً)،
   `SUPABASE_SERVICE_ROLE_KEY`،
   `ADMIN_PASSWORD`
   أو
   `ADMIN_SESSION_SECRET`.
4. ثبّت الحزمة الجديدة:
   `npm install`
   (أضافت `sharp`).
5. شغّل التعبئة الأولى بصورة تجريبية أولاً (بلا أي استدعاء فعلي أو تكلفة):
   `npm run build:catalog:dry`
   — تعرض فقط المجموعات التي ستُدخَل، للتأكد من صحة التجميع قبل الدفع الفعلي.
6. إذا كانت النتيجة منطقية، شغّل التعبئة الفعلية:
   `npm run build:catalog`.
7. راجع كل عنصر جديد قبل ظهوره لبقية المتاجر عبر
   `GET /api/catalog-review.js?action=pending`
   مع رأس
   `x-admin-key`،
   ثم أرسل
   `POST ?action=review`
   بالقرار
   `approve`
   أو
   `reject`
   لكل عنصر (أو
   `bulk-review`
   لعدة عناصر دفعة واحدة).

## ملاحظة مهمة حول العلامات التجارية

لا توجد وسيلة آلية موثوقة 100% لاكتشاف كل علامة أو ملصق خاص بمتجر داخل الصورة، لذلك صُمم النظام بحيث:

- كل عنصر جديد يبدأ بحالة "غير معتمد" ولا يظهر لأي متجر آخر إطلاقاً.
- المعالجة الآلية (إزالة الخلفية + توحيد القياس) تُشغَّل دائماً، وتحاول أيضاً حذف أي شعار أو ملصق أو نص واضح في الصورة.
- ميزة اختيارية
  (`ENABLE_AI_BRAND_PRESCREEN=true`)
  تضيف ملاحظة مبدئية من نموذج رؤية بالذكاء الاصطناعي تساعد المراجع على الفرز بسرعة، لكنها لا تعتمد أي عنصر تلقائياً أبداً — القرار النهائي بشري دائماً.

## تجربة صاحب المتجر

في لوحة المتجر ← المنتجات، يظهر زر "استيراد من مخزن الصور" لمتاجر السوبر ماركت فقط. عند الفتح تظهر بطاقات المنتجات المعتمدة، كل بطاقة فيها: الصورة الموحّدة، حقل الاسم (قابل للتعديل)، حقل سعر اختياري، حقل سعر عرض اختياري، والوحدة. الضغط على "إضافة لمتجري" يحفظ نسخة المتجر الخاصة فوراً بنفس مسار الحفظ الآمن المستخدم عادة عند إضافة منتج يدوياً.
